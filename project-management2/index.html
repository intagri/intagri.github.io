プロジェクトはなぜ失敗するか（後編）

Publish date: Feb 5, 2017
Tags: optimism bias Project Management

前編で紹介したフリュービェアの論文では、プロジェクト失敗の原因の1つとして「楽観バイアス（optimism bias)」について触れると同時に、英国においてこれを抑制するための手法を導入していると簡単に説明しています。では、英国では楽観バイアスにどのように対処しているかを見てみましょう。

英国の楽観バイアス・ガイドライン

2003年、英国財務省は巨大な公共事業の楽観バイアスを抑制する手法を開発・導入しました。これを用いて事業の事前評価（審査）を行わないと予算手当できないこととしています。もともとは、1990年代初頭に行政サービス刷新の一貫として「中央政府による事前評価および事後評価」、いわゆるGreen Bookというガイドラインを策定して公共事業の審査体制を強化しました。このGreen Bookの見直し時に、新たに「楽観バイアス」対策の項目が追加され、optimism biasという補足ガイドラインが発行されました。このガイドライン作成に当たっては、その調査をMott MacDonald社（MM社）に委託しました。調査は、英国での過去20年間の大型公共調達事業のデーターをもとにしています。では、どのように楽観バイアスを削減していくのかを見てみましょう。

MM社はデーターベースの中から選定した50事業を6つのタイプに分け、実績値と計画値との乖離から楽観バイアスを算出、その要因となるプロジェクトのリスクを特定しています。プロジェクトのリスクは5グループ合計21項目に整理され、事業タイプごとにこれらリスクの楽観バイアスへの影響度を算出しています。 まず、事業タイプごとの楽観バイアスの値は以下のように算出されています。

それでは、どのように事業計画を見直すのかをガイドラインの事例（Example 1）に沿って計算してみましょう。

対象事業が非標準的建築で、当初計画の資本支出が現在価値ベース100億円であるとします。まず、表-1から非標準的建築の資本支出の楽観バイアス上限値は51%なので、事業リスクがまったく軽減できないとすると、100億円 x (1 + 0.51) = 151億円が見直しのスタートラインとなります。

次に、表-2のリスクのうち、軽減できるリスクとその軽減率およびそのコストを検討すると、表-3のようになります。

表-3

事業リスク 楽観バイアスへの影響度 軽減率 軽減コスト
コントラクターの能力 5 1.0 0
設計の複雑性 3 1.0 140,000
不適切な事業計画 23 0.4 700,000
乏しい事業関連情報・知見 6 1.0 10,000
事業地特性 1 1.0 40,000

リスク軽減の結果、5 + 3 + (23 x 0.4) + 6 + 1 = 24.2%が軽減できる見込みです。 これにより楽観バイアスは、51 x (1 - 0.242) = 39%となります。軽減に要するコストを含めて計算すると、100 x (1 + 0.39) + (0.0 + 0.14 + 0.7 + 0.01+ 0.04) = 139 + 0.89 = 139.89億円となります。

以上、見ていただいたように100億円の事業の資本支出について楽観バイアスの調整を行うと約140億円となります。さらに、工事期間についても同様の調整をすることになっています。今後、このガイドラインの適用事業が完成していけば、効果検証が行われると期待されますのでその結果が気になるところです。

最後に

プロジェクトはなぜ失敗するのかという問題意識は、10数年前、システム開発に携わった時に遡ります。 納期も予算も当初計画から大幅に増加してしまった開発の最後の1年を担当しました。システム開発未経験者でしたが、なんとかカットオーバー（完成）に漕ぎ着けました。完成後、なぜこんなに遅れ、予算が膨れてしまったのか、気になって仕方がありません。この開発固有の問題だったのか、他社の類似開発ではどうなのか、日本全体ではどうなのか、他の国ではどうなのかという疑問から調べ始めた結果の1つがこの前編、後編となっています。

今一度、表-1の機器・開発（＝システム開発）の楽観バイアス上限値を見てもらうと工期は54%、費用は200%となっています。表-1の中で、最大の上限値です。

システム開発の成否に関する調査は各国で行われており、日本では日本情報システムユーザー協会（JUAS）が毎年調査結果を発表しています。どの国の類似調査においても、当初計画対比で品質（Quality）低下、コスト（Cost）オーバラン、工期（Delivery）遅延が報告されています。成功率は半分くらいもないでしょう。もちろん、長年にわたり、原因とその対策について議論されているのですが、改善の気配はありません。

そういう中で、インフラ事業に限らずさまざまな種類の事業が戦略的な説明（嘘も方便）と勝利の方程式を使って事業の承諾を得ている、もしくは楽観バイアスの影響を受けているというフリュービェアの説明はかなり説得力があります。英国のように、楽観バイアスに対して強制的に見直しをかけるという措置に踏み切った国もあります。

リスクを取るという掛け声のもと、今後も国内外での新規事業、M＆AやAI、IOTなどIT事業が進んでいくのでしょうが、ここで述べたような事業計画策定に内在するリスク、バイアスについて考えてみては如何でしょうか。
